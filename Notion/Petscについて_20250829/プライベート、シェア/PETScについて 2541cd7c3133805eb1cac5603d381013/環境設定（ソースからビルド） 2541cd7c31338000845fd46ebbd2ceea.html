環境設定（ソースからビルド）

PETScの最新版の機能を使用するためには，パッケージ版ではなく，ソースからビルドする必要がある．

取得

下記でGitLabから取得する．

git clone https://gitlab.com/petsc/petsc.git ~/petsc
cd ~/petsc
configure

下記を実行し，最後に Configure state complete と表示され， PETSC_ARCH （例：arch-linux-c-opt）が表示されれば成功

python3 ./configure \
  --with-cc=gcc --with-cxx=g++ --with-fc=gfortran \
  --download-mpich=1 \
  --download-f2cblaslapack=1 \
  --with-debugging=0 \
  --COPTFLAGS="-O3" --CXXOPTFLAGS="-O3" --FOPTFLAGS="-O3"

いずれ，ParMETIS等を使うので，以下のように再configureしておいても良い

cd $PETSC_DIR
make clean                          # 必要なら
python3 ./configure \
  --with-cc=mpicc --with-cxx=mpicxx --with-fc=mpif90 \
  --download-openblas=1 \
  --download-metis=1 --download-parmetis=1 \
  --download-superlu_dist=1 \
  --download-hypre=1 \
  --download-scalapack=1 --download-mumps=1 \
  --download-hdf5=1 \
  --with-debugging=0 \
  --COPTFLAGS="-O3" --CXXOPTFLAGS="-O3" --FOPTFLAGS="-O3"
make all -j$(nproc)
make check
ビルド＆テスト
make all -j"$(nproc)"
make check
環境変数の設定
# configure の出力に従って調整
export PETSC_DIR=$HOME/petsc
export PETSC_ARCH=arch-linux-c-opt   # ← configure が表示したものに合わせる
# pkg-config で使えるように
export PATH="$PETSC_DIR/$PETSC_ARCH/bin:$PATH"
export PKG_CONFIG_PATH="$PETSC_DIR/$PETSC_ARCH/lib/pkgconfig"
# 反映
source ~/.bashrc
# どの petsc.pc を見ているか確認（prefix が PETSC_DIR/PETSC_ARCH のはず）
pkg-config --variable=prefix petsc
# もし /usr/... が出たら PKG_CONFIG_PATH が効いていません
動作確認
#include <petscksp.h>

int main(int argc,char**argv)
{
  PetscCall(PetscInitialize(&argc,&argv,NULL,NULL));
  PetscPrintf(PETSC_COMM_WORLD,"PETSc OK\n");
  PetscCall(PetscFinalize());
  return 0;
}

コンパイルと実行

mpicc example.c -o example $(pkg-config --cflags --libs petsc)
mpiexec -n 2 ./example

実行結果

PETSc OK

並列数2で走らせていて， PETSc OK の文字列が1回しか出力されないのは正常な動作

PetscPrintf(PETSC_COMM_WORLD, "...") は，「そのコミュニケータの代表（通常はrank 0）から1回だけ出力」する関数

この動作確認は，Linux/Ubuntu-22.04/home/takeyama で行っていたが， mnt/d/ 以下でも動作するかをチェックしたところ，動作した．ただし，PETScのビルドなど環境設定は，WSL内で行うことが推奨されている． mnt/d などのWindows側にクローンして行うと，Configureやビルド中に行われるファイルの入出力に非常に時間がかかるため．
